# 캐시 스토리지 정리 가이드

## 현재 상황

캐시 스토리지에 다음 항목이 저장되어 있음:
- webllm
- /config
- wasm
- model

이 중 일부가 손상되었거나 불완전할 수 있어 오류가 발생할 수 있습니다.

## 해결 방법

### 방법 1: 모든 WebLLM 관련 캐시 삭제 (권장)

#### Chrome/Edge:

1. **F12** 키 눌러 개발자 도구 열기
2. **Application** 탭 클릭
3. 왼쪽 메뉴에서 **Cache Storage** 확장
4. 다음 항목들을 **모두 삭제**:
   - `webllm` 관련 항목들
   - `mlc-ai` 관련 항목들
   - 또는 **모든 항목** 삭제 (안전함)

5. **IndexedDB** 확인:
   - 왼쪽 메뉴에서 **IndexedDB** 확장
   - `mlc-ai` 또는 `webllm` 관련 항목 삭제

6. **Clear site data** 버튼 클릭
   - 이렇게 하면 모든 캐시가 삭제됩니다

7. **Ctrl+F5**로 강력 새로고침

#### Firefox:

1. **F12** 키 눌러 개발자 도구 열기
2. **Storage** 탭 클릭
3. **Cache Storage**에서 WebLLM 관련 항목 삭제
4. **IndexedDB**에서 WebLLM 관련 항목 삭제
5. **Ctrl+F5**로 강력 새로고침

### 방법 2: 특정 캐시만 선택적으로 삭제

만약 다른 사이트의 캐시는 유지하고 싶다면:

1. **F12** → **Application** 탭
2. **Cache Storage** 확장
3. 다음 항목들만 선택해서 삭제:
   - `mlc-ai`로 시작하는 항목
   - `webllm`으로 시작하는 항목
   - `model` 관련 항목

### 방법 3: 시크릿 모드로 테스트 (가장 확실)

시크릿 모드는 캐시를 사용하지 않으므로 완전히 깨끗한 상태:

1. **Ctrl+Shift+N** (Chrome/Edge) 또는 **Ctrl+Shift+P** (Firefox)
2. 시크릿 창에서 사이트 접속
3. 모델 다운로드가 정상적으로 진행되는지 확인

## 확인 방법

캐시 삭제 후:

1. **Application** 탭에서 **Cache Storage** 확인
   - WebLLM 관련 항목이 모두 사라졌는지 확인

2. 페이지 **완전 새로고침** (Ctrl+F5)

3. 콘솔에서 다음 메시지 확인:
   ```
   [AI Error Assistant] 모델 로드 시작
   [AI Error Assistant] 모델 로딩 중: X%
   [AI Error Assistant] ✓ WebLLM 엔진 로드 완료
   ```

## 문제가 계속되면

### 1. 브라우저 재시작

브라우저를 완전히 종료하고 다시 시작:
- 모든 탭 닫기
- 브라우저 종료
- 브라우저 재시작

### 2. 다른 브라우저 시도

- Chrome에서 문제가 있으면 Firefox에서 시도
- 또는 Edge에서 시도

### 3. 브라우저 설정에서 저장소 확인

Chrome:
1. 주소창에 `chrome://settings/content/all` 입력
2. 사이트별 저장소 사용량 확인
3. 필요없는 사이트 데이터 삭제

## 예방 방법

정기적으로 캐시 정리:
- 주기적으로 브라우저 캐시 삭제
- 개발 중에는 시크릿 모드 사용 권장

## 참고

- WebLLM은 모델을 캐시에 저장하므로 첫 로드 후 다음 로드는 빠릅니다
- 하지만 캐시가 손상되면 삭제하고 다시 다운로드해야 합니다
- 모델 크기에 따라 다운로드 시간이 걸릴 수 있습니다 (0.5B 모델: 약 300MB)
