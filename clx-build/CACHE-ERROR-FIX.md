# 브라우저 캐시 오류 해결 가이드

## 문제 원인

`net::ERR_FAILED 200 (OK)` 오류는:
- **HTTP 응답은 성공 (200 OK)**이지만
- **브라우저 Cache API에서 저장 실패**

원인:
1. 브라우저 캐시 저장소가 가득 참
2. 브라우저 캐시 크기 제한 초과
3. Cache API 권한 문제

## 빠른 해결 방법

### 방법 1: 브라우저 캐시/저장소 삭제 (가장 확실)

#### Chrome/Edge:
1. **F12** 키 눌러 개발자 도구 열기
2. **Application** 탭 클릭
3. 왼쪽 메뉴에서 **Storage** 확장
4. **Clear site data** 클릭
   - 또는 각 항목별로 삭제:
     - **Cache Storage** → 해당 사이트의 캐시 삭제
     - **IndexedDB** → 해당 사이트의 데이터 삭제
     - **Local Storage** → 해당 사이트의 저장소 삭제
5. 페이지 **새로고침** (Ctrl+F5)

#### Firefox:
1. **F12** 키 눌러 개발자 도구 열기
2. **Storage** 탭 클릭
3. 왼쪽 메뉴에서 **Cache** 선택
4. 해당 사이트의 캐시 삭제
5. **Indexed DB**에서도 해당 사이트 데이터 삭제
6. 페이지 **새로고침** (Ctrl+F5)

### 방법 2: 시크릿 모드에서 테스트

캐시 없이 깨끗한 상태로 테스트:
- **Chrome**: Ctrl+Shift+N
- **Firefox**: Ctrl+Shift+P
- **Edge**: Ctrl+Shift+N

### 방법 3: 브라우저 설정에서 캐시 삭제

#### Chrome:
1. **설정** (chrome://settings/)
2. **개인정보 및 보안** → **인터넷 사용 기록 삭제**
3. **캐시된 이미지 및 파일** 선택
4. **최근 1시간** 또는 **전체 기간** 선택
5. **데이터 삭제**

#### Firefox:
1. **설정** → **개인정보 및 보안**
2. **쿠키 및 사이트 데이터** → **데이터 삭제**
3. **웹 콘텐츠 캐시** 선택
4. **삭제**

### 방법 4: 특정 사이트만 캐시 삭제

1. 주소창에 `chrome://settings/siteData` 입력 (Chrome)
2. 검색창에 사이트 주소 입력 (예: `127.0.0.1`)
3. 해당 사이트 선택
4. **삭제** 클릭

## 확인 방법

캐시 삭제 후:
1. 페이지 **완전히 새로고침** (Ctrl+Shift+R 또는 Ctrl+F5)
2. 콘솔에서 다음 메시지 확인:
   - `[AI Error Assistant] 모델 로드 시작`
   - `[AI Error Assistant] 모델 로딩 중: X%`
   - `[AI Error Assistant] ✓ WebLLM 엔진 로드 완료`

## 추가 해결 방법

### 브라우저 저장소 공간 확인

Chrome에서:
1. 주소창에 `chrome://settings/content/all` 입력
2. 사이트별 저장소 사용량 확인
3. 필요없는 사이트 데이터 삭제

### 브라우저 업데이트

최신 브라우저 버전 사용:
- Chrome/Edge: 최신 버전 사용
- Firefox: 최신 버전 사용

### 하드 디스크 공간 확인

브라우저 캐시는 디스크 공간을 사용하므로:
- C: 드라이브에 충분한 공간 있는지 확인 (최소 1GB 이상 권장)

## 예방 방법

1. **정기적으로 브라우저 캐시 정리**
   - 주기적으로 캐시 삭제

2. **브라우저 설정에서 캐시 크기 제한**
   - Chrome: `chrome://settings/system` → 캐시 크기 제한

3. **시크릿 모드 사용**
   - 테스트 시 시크릿 모드 사용 (캐시 없음)

## 문제가 계속되면

1. **다른 브라우저에서 테스트**
   - Chrome에서 문제가 있으면 Firefox에서 시도

2. **WebLLM 공식 이슈 확인**
   - https://github.com/mlc-ai/web-llm/issues

3. **브라우저 콘솔의 상세 오류 확인**
   - 오류 메시지 전체 내용 확인

## 요약

**가장 확실한 해결 방법:**
1. F12 → Application/Storage 탭
2. Clear site data 또는 Cache Storage 삭제
3. Ctrl+F5로 강력 새로고침

이렇게 하면 90% 이상의 캐시 관련 문제가 해결됩니다!
